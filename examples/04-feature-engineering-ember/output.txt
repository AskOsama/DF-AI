STEP 6 — LEAKAGE TRAP (real demonstration, 5-fold cross-validation)
========================================================================
  We add a LEAKY feature 'source_folder_id': every benign file gets
  id=0, every packed file gets id=1 -- exactly the 'folder named
  evidence' shortcut. In a real case the analyst's intake script wrote
  benign and suspicious files to different folders, so the folder id
  is PERFECTLY correlated with the label. The model can cheat.

  We compare HONEST vs LEAKY feature sets with 5-fold CV (stable est.)

  WITHOUT leaky feat. (honest): accuracy=0.9417  AUC=0.9889
  WITH leaky feature          : accuracy=1.0000  AUC=1.0000
     -> leaky feature ranked #1 of 32 in importance (it dominates)
  INFLATION from leakage      : accuracy +0.0583   AUC +0.0111

  LESSON: the leaky model looks great offline but learned the FOLDER,
          not the FILE. On genuinely unknown evidence -- where every
          file is 'unsorted' -- the shortcut is gone and it collapses.
          -> drop folder/timestamp/source-feed metadata before training.
