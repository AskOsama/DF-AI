مجموعة البيانات والتنزيل

مجموعة بريد Enron (توزيع CMU). نحو 500,000 رسالة بريد مؤسسية حقيقية من نحو 150 موظفاً كباراً في Enron، أفرجت عنها FERC أثناء تحقيق 2001–2004 — مجموعة الكشف الإلكتروني/الجنائية النموذجية، والمختبَر المعياري لبحوث استرجاع المعلومات.

الخاصيةالقيمة
الترخيصعامة، قابلة للتنزيل بحرية للبحث؛ تطلب CMU فقط احترام خصوصية المعنيين. (لا ترخيص SPDX رسمي — عُدّها عامة-للبحث.)
الحجم~1.7 جيجابايت مضغوطة؛ ~2.6 جيجابايت غير مضغوطة؛ ~0.5 مليون رسالة.
الصيغةشجرة Maildir — ملف نصّي RFC-822 لكل رسالة (maildir/<user>/<folder>/<n>) بترويسات قياسية + متن.
المصدرcs.cmu.edu/~enron (نسخة مايو 2015)
 
wget https://www.cs.cmu.edu/~enron/enron_mail_20150507.tar.gz
tar -xzf enron_mail_20150507.tar.gz          # creates ./maildir/
pip install rank-bm25 nltk                    # BM25 ranking + tokeniser
python -m nltk.downloader stopwords punkt
# persistent on-disk index alternative: pip install Whoosh

ملاحظة عن النطاق. الـ2.6 جيجابايت هي الأثر الذي يفهرسه الطالب فعلاً؛ والشِفرة نفسها تتوسّع إلى مجموعة قرص محجوز بحجم تيرابايتات كالتي يصفها الجزء الرابع — التغيير الوحيد هو تجزئة الفهرس. (ولقصة قرص محجوز حرفية بدلاً من ذلك، يُعدّ سيناريو M57-Patents من ديجيتال كوربورا البديل.)

السيناريو الجنائي

على محلّل كشف إلكتروني تحت حجز قضائي أن يُبرِز كل رسالة داخلية تناقش كيانات الأغراض الخاصة خارج الميزانية وأدوات التحوّط Raptor/LJM، مُرتَّبةً بالصلة، ليراجع المحامي أكثر المستندات دلالةً أولاً. وبدل مسح 500 ألف ملف خطّياً، يبني المحلّل فهرساً معكوساً مرة واحدة، ثم يُصدِر استعلامات مُرتَّبة في أجزاء من الثانية (raptor, LJM, "special purpose entity") ويتمحور حول أعلى النتائج.

الترميز: الفهرس المعكوس

تصير رسائل RFC-822 الخام فهرساً معكوساً عبر أربع مرورات:

المرورما يحدث
التحليليقرأ email.parser كل ملف maildir؛ احتفِظ بـMessage-ID معرِّفَ مستند، واحتفِظ بـFrom/To/Subject/Date حقولاً + المتن محتوىً قابلاً للبحث.
التقطيعقسّم المتن+الموضوع على حدود غير أبجدية-رقمية (\w+)؛ اختيارياً أبقِ رموز عناوين البريد سليمة.
التسويةتصغير الأحرف ← إسقاط كلمات الوقف ← تجذير Porter/Snowball (hedging→hedg) ← إسقاط الرموز <حرفين.
القوائمterm → [(doc_id, term_freq, [positions])]؛ المواضع تتيح استعلامات العبارة/التجاور. وتربط doc_store موازِيةٌ doc_id → {subject, from, date, snippet} للعرض.

نموذج الترتيب: BM25 (Okapi) مُقيِّماً أساسياً (k1=1.5, b=0.75)، مع TF-IDF خط أساس أبسط. يُفضَّل BM25 لأنه يُشبِع تكرار المصطلح ويُسوّي الطول — وهو ما يهمّ مع تباين أطوال البريد الجامح في المجموعة.

الخوارزمية وبنى البيانات

  • الفهرس المعكوس: dict[str, list[Posting]] (مصطلح ← قائمة قوائم) مع جدول أطوال المستندات ومتوسط طول المجموعة لأجل BM25.
  • التقييم: اجمع BM25 فوق مصطلحات الاستعلام؛ رتّب المستندات تنازلياً بالدرجة. وتنطبق استراتيجيات بحث الجزء الرابع على الاسترجاع — القوة الغاشمة = مسح خطّي لكل 500 ألف ملف (ما نتجنّبه)؛ الجشِع = توسيع أعلى القوائم درجةً أولاً؛ الجشِع-إبسيلون/العشوائي = أخذ عيّنات من نتائج أدنى أحياناً للتنويع أثناء المراجعة.
  • المكتبات: rank_bm25 (Apache-2.0) لخط معالجة شفاف من الصفر؛ scikit-learn TfidfVectorizer لخط الأساس؛ Whoosh لفهرس حقيقي على القرص مع مُحلِّل استعلام؛ Tantivy / Elasticsearch للتوسّع الإنتاجي.

التنفيذ

  1. نزّل واستخرج المجموعة؛ ثبّت rank-bm25 + nltk.
  2. الاستيعاب: امشِ maildir/، حلّل كل ملف، ابنِ doc_store[doc_id] = {from,to,subject,date,body}.
  3. المعالجة المسبقة: تصغير ← تقطيع ← إسقاط كلمات الوقف ← تجذير ← tokenized_corpus.
  4. بناء الفهرس: BM25Okapi(tokenized_corpus) (أو مخطّط Whoosh على القرص).
  5. الاستعلام: عالِج الاستعلام بالطريقة نفسها ← get_scores ← argsort ← أعلى-k.
  6. العرض: اربط معرّفات أعلى-k بـdoc_store للمُرسِل/التاريخ/الموضوع + مقتطف كلمة-في-سياق؛ نقّح بقيود العبارة ومرشّحات from:.
import os, email
from rank_bm25 import BM25Okapi
from nltk.corpus import stopwords
from nltk.stem import SnowballStemmer

stop, stem = set(stopwords.words('english')), SnowballStemmer('english')
def prep(text):
    return [stem.stem(w) for w in __import__('re').findall(r'\w+', text.lower())
            if w not in stop and len(w) > 1]

docs, corpus = [], []
for root, _, files in os.walk('maildir'):
    for fn in files:
        msg = email.message_from_file(open(os.path.join(root, fn),
                                           errors='ignore'))
        body = msg.get_payload() if isinstance(msg.get_payload(), str) else ''
        docs.append({'from': msg['From'], 'date': msg['Date'],
                     'subject': msg['Subject'], 'body': body})
        corpus.append(prep((msg['Subject'] or '') + ' ' + body))

bm25 = BM25Okapi(corpus)
scores = bm25.get_scores(prep('raptor LJM special purpose entity'))
top = sorted(range(len(scores)), key=lambda i: scores[i], reverse=True)[:10]

المخرجات النهائية

قائمة نتائج مُرتَّبة يراجعها المحلّل من الأعلى — يُظهِر كل صفّ درجة الصلة وبيانات وصفية بنيوية ومقتطف كلمة-في-سياق:

 2    21.6   2001-10-02  richard.causey@enron.com  SPE accounting treatment   "…special purpose entity does not require consolidation…"
 3    19.8   2001-08-23  …                         Re: LJM2 capital           "…LJM2 funding for the special purpose entities…"

يُبرهن المُخرَج الادّعاء الجنائي الجوهري: ابنِ الفهرس مرة واحدة، ثم استرجِع نتائج مُرتَّبة في أجزاء من الثانية — لتمكين المحامي من مراجعة أكثر المستندات دلالةً أولاً بدل مسح 500 ألف ملف خطّياً.
