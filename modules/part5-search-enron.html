Search Engines — the Agent's Run on 80,000 Real Enron Emails

The executed outcome behind the worked prompt for search. The pipeline really built a BM25 inverted index over the Enron corpus and ran ranked e-discovery queries; below is the agent's loop — Approach → Action → Observation → Done — with the genuine captured output. The indexing model and ranking are on the Part 4 dataset page.

What was run

Forensic question: under a litigation hold, surface every internal email discussing the off-balance-sheet special-purpose entities and the Raptor/LJM hedges, ranked by relevance so counsel reviews the most probative documents first. The agent builds a BM25 inverted index over 80,000 real Enron messages, runs a broad query, then refines it. Verified outcome: sub-second ranked retrieval surfacing genuine valuation memos (Kaminski's "LJM/Raptor valuations"), with query refinement and a custodian pivot to Fastow.

Plan-phase prompt

What the analyst types to make the agent plan the search (no code yet):

You are an e-discovery analyst supporting a litigation hold on the Enron matter.
Goal: surface every internal email discussing the off-balance-sheet
special-purpose entities and the Raptor / LJM hedging vehicles, RANKED by
relevance so counsel reviews the most probative documents first.

Produce a PLAN only (no execution). Specify:
  1. Dataset acquisition: source, license, size, maildir/RFC-822 format. Note
     key custodians (and verify whether Fastow/Causey mailboxes even exist).
  2. Ingestion: how you parse each message, which fields you keep, and how you
     bound memory/time (message cap; walk priority custodians first).
  3. Index design: tokenization (\w+), normalization (lowercase, stopwords,
     Snowball stem), postings term -> [(doc_id, tf, positions)], ranking model.
     Justify BM25 (k1=1.5, b=0.75) over TF-IDF for email.
  4. Query design: an initial broad query + >=2 refinements (tighter accounting
     terms; a sender/custodian filter).
  5. Output: a ranked top-N table with date/from/subject + a KWIC snippet.
  6. The human-in-the-loop gate: the ranking is a starting point for REVIEW
     ORDER, not a coverage/responsiveness guarantee.
Return the plan as numbered steps with the data structures named.

Execute-phase prompt

What the analyst types to run it:

Execute the e-discovery search plan now, end to end, with real output.
  1. Build the venv; install rank-bm25 + nltk; download stopwords + punkt.
  2. Download and extract the Enron corpus to data/maildir/. If full extraction
     is too slow, extract the scenario custodians only and SAY SO.
  3. Implement run.py: parse with email.parser, preprocess, build
     BM25Okapi(tokenized_corpus, k1=1.5, b=0.75), confirm key custodians are
     indexed (print counts).
  4. Run "raptor LJM special purpose entity mark to market" and print the TOP 10
     as rank/score/date/from/subject + a ~120-char KWIC snippet. Real values only.
  5. Refine: (a) a tighter accounting query, (b) a sender filter From~"fastow".
     Print the new tops and how the ranking changed.
  6. python3 run.py | tee output.txt. Report indexed-message count, the real top
     hits, and any deviation. Do not fabricate any score or sender.

The run: Approach → Action → Observation

Each Observation is verbatim from the real output.txt.

Step 1 — Ingest and build the index

Approach. Parse the maildir (priority custodians first, capped for time/memory), then build the BM25 inverted index.

Action:  ingest(maildir, cap=80000); build_bm25_index()

Observation:
    Parsed 80,000 messages.  Key custodians present:
       kaminski-v 28465 | kean-s 19245 | lay-k 5937 | skilling-j 4139 | buy-r 2429 ...
    80,000 docs, 17,414,728 tokens, 145,360 unique terms.
    BM25 build time 3.7s | avgdl 217.7 | k1=1.5, b=0.75

Step 2 — Broad query (see what surfaces)

Approach. Start broad on the scenario terms to see what the corpus surfaces — and what noise comes with it.

Action:  search("raptor LJM special purpose entity mark to market")   [ranked in 0.176s]

Observation (top hits):
   4  26.17  j.kaminski@enron.com   FW: LJM/Raptor valuations
  10  23.48  j.kaminski@enron.com   LJM/Raptor valuations
   1  29.09  pr <.palmer@enron.com> FW: Erroneous press     <- NOISE (PR thread, repeats "LJM")
   2  29.01  mark.koenig@enron.com  FW: Erroneous press     <- NOISE
   -> true gold (Kaminski valuations) is crowded by high-frequency PR chatter.

Step 3 — Refine with accounting vocabulary

Approach. The PR chatter crowds out the substantive memos; tighten with disclosure/accounting terms the probative docs share but the press chatter does not.

Action:  search("raptor LJM hedge off balance sheet special purpose entity
                 accounting restatement")

Observation:
   1  42.97  chairman.ken@enron.com  SEC Information/Earnings Restatement
      KWIC: ...related party and off-balance sheet transactions...
   -> ranking shifts hard: only 3/10 carry over from Q1, 7 are new; the
      disclosure/restatement core now dominates.

Step 4 — Pivot to a custodian (sender filter)

Approach. Pivot from topic to custodian — find anything authored by Fastow, even though his mailbox isn't in the corpus.

Action:  search("raptor LJM special purpose entity mark to market", sender~"fastow")

Observation:
   1  16.65  andrew.fastow@enron.com  FW: Monday Meeting
      KWIC: ...it looks like LJM is be[ing favoured]...
   -> 5 real Fastow messages (sitting in OTHER custodians' boxes); custodian
      pivot succeeds despite the missing mailbox.

>>> Human-in-the-loop gate. BM25 gives review order, not a responsiveness ruling. The analyst decides where the responsive set ends, whether to broaden custodians, and whether near-duplicate forwarded copies count once or many times. The score never closes the matter.

Done — the result

Index: 80,000 messages, 145,360 terms, sub-second retrieval (0.176s).
Q1 broad  : top score 29.09 — Kaminski "LJM/Raptor valuations" + PR noise.
Q2 refined: top score 42.97 — Ken Lay "SEC Information/Earnings Restatement";
            7/10 hits new vs Q1 (tighter terms re-rank the set).
Sender    : From~"fastow" -> 5 real Fastow originals (top "FW: Monday Meeting").
=> a ranked review queue counsel works top-down, instead of scanning 500k files.

The headline: build the index once, then retrieve ranked hits in milliseconds. The agent's value is iterating — search, read, refine, pivot — to push probative documents to the top of the review queue.

Guidance: reading the outcome

  • Ranking is review order, not coverage. The top-N is where to start reading, not a guarantee of completeness. The human decides where the responsive set ends.
  • Refinement is the skill. The broad query surfaced PR noise at #1–3; adding accounting terms re-ranked the substantive disclosure memos to the top. Teaching point: iterate the query, don't trust the first list.
  • Failure mode — no semantics. Keyword/stem ranking has no meaning model: a memo saying "the structured-finance vehicles we used to keep debt off the books" without ever typing raptor, LJM, or SPE ranks low or misses; high-frequency mentions in irrelevant PR chatter rank spuriously high (exactly the #1–3 in Q1). Mitigation: query expansion / synonyms, phrase & proximity queries, and ultimately human review.

Honest deviation (teaching note). The fastow-a and causey-r mailboxes do not exist in the public CMU/FERC release; Fastow still surfaces via the sender filter because his mail sits in others' boxes. Indexing was capped at 80,000 messages (priority custodians walked first) because full extraction stalled on the Windows mount. Near-duplicate forwarded copies are left in, to show students the sent/inbox artefact.
