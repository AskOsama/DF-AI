مجموعة البيانات والتنزيل

GovDocs1 (ديجيتال كوربورا). نحو مليون ملف حقيقي (PDF, DOC, XLS, PPT, HTML, JPG, TXT, CSV…) جُمعت من خوادم .gov — مجموعة الاختبار المعيارية بحكم الأمر الواقع في الجنايات الرقمية الأكاديمية. تُشحَن على هيئة subset0.zip … subset9.zip، كلٌّ منها 1000 ملف، ليتمكّن الطالب من التجريب على مجموعة فرعية واحدة.

الخاصيةالقيمة
الترخيصمتاحة بحرية وقابلة لإعادة التوزيع للبحث (برنامج بيانات أمازون المفتوحة). مؤكَّدة مفتوحة الوصول.
الحجمالمجموعة الكاملة ~مئات الجيجابايت؛ subsetN.zip واحدة ≈ عشرات إلى مئات الميجابايت. استخدم مجموعة فرعية واحدة.
الصيغةملفات مكتبية/وِب/صور متنوّعة، مسمّاة NNN/NNNNNN.ext؛ يُشحَن معها تقرير فحص برمجيات خبيثة (إشارة قيمة مفيدة).
المصدرdigitalcorpora.org · S3: s3://digitalcorpora/corpora/files/govdocs1/
 
# list the bucket first to confirm the exact subset path
aws s3 ls s3://digitalcorpora/corpora/files/govdocs1/ --no-sign-request
# then fetch one subset (no AWS account needed)
curl -L -O https://downloads.digitalcorpora.org/corpora/files/govdocs1/subsets/subset0.zip
unzip subset0.zip -d ./corpus
pip install ortools deap numpy python-magic   # OR-Tools + DEAP (NSGA-II)

السيناريو الجنائي

صورة قرص من محطة عمل محجوزة جرى استخراج محتواها، فأنتجت 1000 ملف من مجموعة GovDocs1 فرعية واحدة (عُدّ كلاً منها أثراً مُستخرَجاً). لدى محلّلَين معاً 16 ساعة-محلّل قبل موعد المحكمة. بأيّ ترتيب، وعلى طابور مَن، ينبغي فحص الملفات لتعظيم القيمة الاستدلالية المتوقَّعة ضمن ميزانية الوقت؟ جداول البيانات والمستندات عالية القيمة لكل دقيقة؛ والوسائط الكبيرة بطيئة؛ وقليل من الملفات المُعلَّمة كخبيثة عالية القيمة لكنها مكلِفة.

الترميز: الملفات ← عناصر

يصير كل ملف مُستخرَج سجلاً بـتكلفة وقيمة شفافتين:

السمةكيف تُحسَب (بثمن بخس وقابلية تكرار)
cost_minutesbase[ext] + k · size_MB، مثلاً base = {xls:8, doc:6, pdf:5, ppt:5, html:3, jpg:4, txt:2}، k ≈ 2.
value_scoreأولوية نوع الملف + مكافأة تقرير الفحص الخبيث + إصابات تعابير نمطية/عشوائية معلوماتية (بريد، أنماط بطاقات/IBAN، عشوائية عالية = تشفير/إخفاء محتمل).
زوج العنصر(id, ext, size, cost_minutes, value_score)
الهدف الثاني (باريتو)coverage = عدد أنواع الملفات المتمايزة المَطروقة — الشمولية مفاضَلةً مع القيمة والوقت.

أبقِ معادلة القيمة قابلة للإفصاح. كل وزن (أولويات الأنواع، مكافأة الخبيث، مجموعة التعابير النمطية) رقم يمكنك الدفاع عنه في المحكمة. وفي اللحظة التي تصير فيها "القيمة" درجة صندوق أسود، يفقد ترتيب الفرز قابليته للتفسير — وهو الفخّ نفسه الذي يحذّر منه الجزء الرابع بشأن تعلّم الآلة.

الخوارزمية وبنى البيانات

  • خطوط الأساس (من الجزء الرابع): جشِع (ترتيب بنسبة القيمة/التكلفة)، جشِع-إبسيلون، عشوائي، وقوة غاشمة (على شريحة ≤20 عنصراً فقط) — الاستراتيجيات الواجب التفوّق عليها.
  • حقيبة الظهر أحادية الهدف: السعة = دقائق المحلّل، تعظيم مجموع القيمة. OR-Tools KnapsackSolver؛ البيانات: متجهات قيمة/تكلفة NumPy، وطابور أولوية heapq لخط الأساس الجشِع.
  • محلّلان: نموذج حقيبة ظهر متعدّدة / CP-SAT يُسنِد الملفات إلى صندوقَي 8 ساعات، مع تعظيم القيمة الكلية ضمن كل ميزانية.
  • باريتو متعدّد الأهداف: DEAP NSGA-II (tools.selNSGA2، tools.ParetoFront) فوق الأهداف (maximise value, maximise coverage, minimise time)؛ وIndividual هو متجه شمول/استبعاد ثنائي.

التنفيذ

  1. نزّل مجموعة فرعية واحدة وفُكّ ضغطها في ./corpus.
  2. استخرج السمات: امشِ المجموعة، واحسب cost_minutes وvalue_score لكل ملف ← items.csv.
  3. شغّل خطوط الأساس الأربعة لميزانية 16 ساعة واحدة؛ سجّل القيمة الكلية.
  4. احلل الأمثل العددي (ميزانية واحدة) وتقسيم المحلّلَين (CP-SAT)؛ قارن بخطوط الأساس.
  5. شغّل NSGA-II لجبهة باريتو قيمة/شمول/وقت؛ اجمع الخطط غير المُهيمَن عليها.
import numpy as np
from ortools.algorithms.python import knapsack_solver

# items.csv already has cost_minutes (int) and value_score (int) per file
values  = items["value_score"].to_list()
weights = [items["cost_minutes"].to_list()]      # one dimension: minutes
capacity = [16*60]                                # 16 analyst-hours

s = knapsack_solver.KnapsackSolver(
    knapsack_solver.SolverType.KNAPSACK_MULTIDIMENSION_BRANCH_AND_BOUND_SOLVER,
    "triage")
s.init(values, weights, capacity)
best = s.solve()
chosen = [i for i in range(len(values)) if s.best_solution_contains(i)]
print(f"selected {len(chosen)} files, total value {best}")

# greedy baseline for comparison
order = sorted(range(len(values)),
               key=lambda i: values[i]/weights[0][i], reverse=True)

المخرجات النهائية

يتلقّى المحلّل شيئين:

  • ترتيب فحص مُرتَّب مقسوماً إلى طابورَي المحلّل A والمحلّل B، كلٌّ يحترم ميزانيته (8 ساعات)، مع الدرجة الكلية للدليل المتوقَّع وجدول يبيّن تفوّق أداة الأمثلة على الجشِع / الجشِع-إبسيلون / العشوائي.
  • جبهة باريتو من 4–5 خطط بديلة قابلة للدفاع — مثلاً "عالية القيمة / ضيّقة" مقابل "واسعة الشمول / أقل قيمة" — ليختار القائد المفاضلة المناسبة للقضية.
