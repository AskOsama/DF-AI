المعرفة والاستدلال — استنتاج المسارات على شبكة طيران

مسارات ومطارات OpenFlights. مخطّط عالمي يضمّ نحو 67,663 مسار طيران مجدوَلاً بين نحو 3,321 مطاراً على 548 شركة، إضافةً إلى دليل بأكثر من 10,000 مطار مع بيانات المدينة/البلد/الموقع. صغير، بصيغة CSV بسيطة، ويُحمَّل كاملاً في الذاكرة بسهولة.

السيناريو الجنائي

يدّعي المشتبه به X أنه كان في دبي ليلة الحادثة، ومن ثَمّ لم يكن بإمكانه أن يكون في بوغوتا (BOG) حيث وقعت الجريمة. تضع سجلّات الحدود مغادرته من لندن هيثرو (LHR) قبل نحو 28 ساعة، ولا توجد خدمة مباشرة LHR→BOG في مخطّط المسارات. هل توجد أيّ سلسلة رحلات متّصلة (≤3 وصلات) تتيح لـ X السفر LHR → BOG ضمن النافذة الزمنية — وعبر أيّ محاور؟ إن وُجدت "نعم، عبر المحور H" دحضت حجّة الغياب، ويُسمّي المحرّك الرحلات المحدّدة التي تجعلها ممكنة.

الترميز: الحقائق والقواعد

يصير كل صفّ CSV حقيقة أرضية؛ وتُعرَّف قابلية الوصول بـقواعد.

# Facts extracted from the CSVs
airport('LHR', 'London', 'United Kingdom').
airport('BOG', 'Bogota', 'Colombia').
flight('BA', 'LHR', 'MAD').        % from routes.dat
flight('AV', 'MAD', 'BOG').

# Rules (the inference engine)
reachable(A, B, [A,B])      :- flight(_, A, B).               % one hop
reachable(A, C, [A|Path])   :- flight(_, A, B),               % transitive
                               reachable(B, C, Path),
                               \+ member(A, Path).             % no cycles
contradicts_alibi(S, O, C, Path) :- claimed_origin(S, O),
                                     reachable(O, C, Path),
                                     length(Path, L), L =< 4.  % ≤3 connections

الفكرة نفسها في pyDatalog (المكتبة الأساسية)، مع تحميل الحقائق في حلقة من routes.dat:

from pyDatalog import pyDatalog
pyDatalog.create_terms('flight, reachable, X, Y, Z, W')
+ flight('BA', 'LHR', 'MAD'); + flight('AV', 'MAD', 'BOG')
reachable(X, Y) <= flight(Z, X, Y)                  # base case
reachable(X, Y) <= flight(Z, X, W) & reachable(W, Y) # recursive case
print(reachable('LHR', 'BOG'))                       # query

الخوارزمية وبنى البيانات

  • استراتيجية الاستدلال: التسلسل الخلفي (موجَّه بالهدف: "هل BOG قابلة للوصول من LHR؟") — الخيار الطبيعي، يستكشف فقط المسارات ذات الصلة بالهدف. ونسخة التسلسل الأمامي (تجسيد الإغلاق المتعدّي كاملاً ثم البحث) تمرين تباين جيّد.
  • مخطّط المعرفة: مخطّط متعدّد موجَّه — العقد = مطارات (IATA)، الأضلاع = رحلات (ضلع لكل خدمة طيران). عملياً networkx.MultiDiGraph؛ تنطبق صفوف routes.dat على الأضلاع 1:1، ويوفّر airports.dat سمات العقد.
  • الاستدلال ≡ بحث: قابلية الوصول التراجعية مع تجنّب الدورات هي بحث محدود بالعمق/العرض؛ تحقّق متقاطعاً بـnetworkx.all_simple_paths(G, src, dst, cutoff=3) وhas_path.
  • المكتبات: pyDatalog (LGPL) لمحرّك القواعد؛ networkx (BSD) للمخطّط وخوارزميات المسار؛ SWI-Prolog اختيارياً لنسخة Prolog خالصة، أو Experta/CLIPS لعرض تسلسل أمامي بـRETE.

التنفيذ

  1. نزّل routes.dat وairports.dat.
  2. حلّل الحقائق: المطارات ← airport(IATA, City, Country)؛ المسارات ← flight(Airline, Src, Dst)، متجاوزاً صفوف رموز \N.
  3. أكّد كل حقائق flight/3 في pyDatalog (و/أو ابنِ MultiDiGraph).
  4. عرّف قواعد reachable وميزانية الوصلات (≤3 وصلات).
  5. استعلِم reachable('LHR','BOG')؛ استخرج المسار الشاهد وحقائق flight/3 على كل ضلع.
  6. تحقّق متقاطعاً بـnetworkx.all_simple_paths(G,'LHR','BOG',cutoff=3).
import csv, networkx as nx

G = nx.MultiDiGraph()
with open('data/airports.dat', encoding='utf-8') as f:
    for r in csv.reader(f):
        iata = r[4]
        if iata != '\\N':
            G.add_node(iata, city=r[2], country=r[3])
with open('data/routes.dat', encoding='utf-8') as f:
    for r in csv.reader(f):
        src, dst, airline = r[2], r[4], r[0]
        if src in G and dst in G:
            G.add_edge(src, dst, airline=airline)

paths = list(nx.all_simple_paths(G, 'LHR', 'BOG', cutoff=3))
print(paths[0])     # -> ['LHR', 'MAD', 'BOG']

المخرجات النهائية

الحقيقة المستنتَجة: contradicts_alibi('SubjectX','LHR','BOG',['LHR','MAD','BOG']) — صحيحة. يوجد مسار بوصلة واحدة عبر مدريد، ضمن النافذة الزمنية بأريحية، فحجّة دبي متناقضة مع شبكة الطيران.

والعائد التربوي هو السلسلة القابلة للتدقيق — لا احتمال، بل مسار كل ضلع فيه خدمة طيران حقيقية مُسمّاة، واشتقاقه يستشهد بالقاعدة التراجعية المطبَّقة عند كل خطوة:

GOAL: reachable('LHR','BOG')
 ├─ RULE  reachable(X,Y) <= flight(_,X,W) & reachable(W,Y)
 │   ├─ FACT  flight('BA','LHR','MAD')     [routes.dat: BA,…,LHR,…,MAD]
 │   └─ GOAL  reachable('MAD','BOG')
 │        └─ RULE reachable(X,Y) <= flight(_,X,Y)
 │             └─ FACT flight('AV','MAD','BOG')  [routes.dat: AV,…,MAD,…,BOG]
 └─ DERIVED PATH: LHR --BA--> MAD --AV--> BOG   (connections = 1 ≤ 3)

كيف يشغّل الوكيل هذا: راجع المطالبات التطبيقية في الجزء الخامس لطلب المحلّل ولأثر ReAct الذي يستعلم قاعدة المعرفة ويُبرِز حقائق الطيران الكامنة.
