هندسة السمات — فرز البرمجيات الخبيثة استاتيكياً من البايتات الخام

أوضح قسم إعداد البيانات النقطة الجوهرية: لا يُسلَّم الملف إلى المصنِّف بوصفه ملفاً — بل بوصفه قائمة سمات (الحجم، العشوائية المعلوماتية، بايتات الترويسة، …)، والسمات الرديئة تُغرِق حتى الخوارزمية المثالية. وEMBER هو المثال النموذجي على ذلك تماماً: ملف Windows تنفيذي يُختزَل إلى متجه سمات بـ2,380 بُعداً. تمشي هذه الصفحة بخط البايتات الخام ← السمات ← الحكم، بما في ذلك فخّ التسرّب الواجب تجنّبه.

مجموعة البيانات والتنزيل

EMBER 2018 (سمات الإصدار 2) — مرجع البرمجيات الخبيثة من Elastic. سمات استاتيكية مُهندَسة مستخرَجة من نحو مليون ملف Windows PE (خبيث / حميد / غير مُعنوَن)، مُصدَرة مع شِفرة الاستخراج — ليرى الطلاب بالضبط كيف تصير البايتات الخام سمات. إنها قصة "الأثر الخام ← متجه السمات" النموذجية.

الخاصيةالقيمة
الترخيصملفات البيانات MIT؛ شِفرة الاستخراج AGPL-v3. البيانات التي تصنّفها قابلة للاستخدام بحرية. مؤكَّدة مفتوحة الوصول.
الحجم~مليون عيّنة؛ المصفوفة المُتَّجهة ~1M × 2,380 float32 (الأرشيف عدة جيجابايت).
الصيغةسمات خام لكل عيّنة بصيغة JSONL + مُتَّجِه يُنتِج خرائط ذاكرة ثنائية X_*.dat / y_*.dat.

السيناريو الجنائي

يستخرج محلّل ملف Windows تنفيذياً مجهولاً من مساحة غير مخصَّصة — بلا توقيع، بلا حكم مضادّ للفيروسات. هل هو حميد أم خبيث؟ — يُجاب دون تشغيله، من خصائص بايتية وبنيوية قابلة للقياس فحسب. وعلى نطاق واسع، يرتّب التقييم نفسه 10,000 ملف تنفيذي مُستخرَج ليراجع الإنسان الأكثر اشتباهاً أولاً.

الترميز: متجه السمات

يصير كل ملف PE خام متجهاً ثابتاً بـ2,380 بُعداً عبر تسع مجموعات (سمات الإصدار 2):

المجموعة (الأبعاد)تُحسَب من البايتات الخام
مدرّج البايتات (256)عدّ كل قيمة بايت 0x00–0xFF، مُسوّى — التركيب الإجمالي.
مدرّج عشوائية البايتات (256)عشوائية شانون بنافذة منزلقة؛ المناطق العالية تشير إلى التغليف/التشفير.
إحصاءات السلاسل (109)عدد السلاسل القابلة للطباعة، متوسط الطول، عشوائية توزّع المحارف، علامات المسار/URL/السجلّ.
معلومات عامة (10)الحجم، الحجم الافتراضي، أعداد الاستيراد/التصدير، وجود تنقيح/TLS/توقيع.
معلومات الترويسة (62)نوع آلة/سحر COFF، الطابع الزمني، الخصائص؛ النظام الفرعي، إصدار الرابط.
معلومات المقاطع (305)اسم وحجم وعشوائية وأعلام وصول كل مقطع — مُجزّأة سماتياً.
الاستيرادات (1,280)مكتبات DLL + دوال واجهات (مثل VirtualAlloc, WriteProcessMemory) — مُجزّأة.
التصديرات (128)أسماء الدوال المُصدَّرة — مُجزّأة سماتياً.
أدلّة البيانات (30)حجم وعنوان أدلّة بيانات PE الخمسة عشر.

فخّ التسرّب. يتسرّب الطابع الزمني في ترويسة PE تحت تقسيم EMBER الزمني (التدريب قبل عتبة، الاختبار بعدها): إذ يستطيع النموذج تعلّم "طابع أحدث ⇐ صنف X"، وهو اختصار لن يعمّم على ملفات مجهولة فعلاً. ويسري الخطر نفسه على أيّ بيانات وصفية لاسم/مسار/مصدر الملف — وهو عين اختصار "مجلّد باسم أدلّة" الذي يحذّر منه الجزء الرابع. أسقِط السمات المشتقّة من الطابع الزمني أو دقّق فيها.

الخوارزمية وبنى البيانات

  • مصفوفة السمات: صفّ لكل ملف = الأبعاد 2,380 أعلاه، مُسلسَلة بترتيب المجموعات الثابت؛ المجموعة هي X بشكل (n, 2380) (خريطة ذاكرة float32) والعناوين y (0 حميد، 1 خبيث، -1 غير مُعنوَن ← يُسقَط).
  • المصنِّف: أشجار LightGBM المُعزَّزة بالتدرّج (MIT، LGBMClassifier) — خط أساس EMBER الرسمي (AUC ≈ 0.999). تتعامل أشجار GBDT مع السمات المتغايرة وغير المُقيَّسة والمُجزّأة أصلاً دون تسوية. بديل: scikit-learn RandomForestClassifier.
  • قابلية التفسير: model.feature_importance() مُسقَطة على المجموعات التسع؛ وSHAP لكل عيّنة على الأثر الوحيد قيد المراجعة.

التنفيذ

  1. نزّل واستخرج EMBER؛ pip install lightgbm وحزمة ember المساعِدة.
  2. اتّجِه السمات الخام بصيغة JSONL ← X_*.dat / y_*.dat (هذه الخطوة هي عرض هندسة السمات).
  3. حمّل وأسقِط الصفوف غير المُعنوَنة (y == -1).
  4. درّب LightGBM؛ اطبع أهمّيات السمات مُسقَطة على المجموعات.
  5. فحص التسرّب: أعِد التدريب مستبعِداً سمات الطابع الزمني؛ قارن AUC الاختبار عبر التقسيم الزمني.
  6. صنّف المجهول المُستخرَج؛ فسّر العيّنة الوحيدة بـSHAP.
import ember
ember.create_vectorized_features("ember2018/")          # raw JSONL -> matrix
X, y = ember.read_vectorized_features("ember2018/", "train")
X, y = X[y != -1], y[y != -1]                            # drop unlabeled

model = ember.train_model("ember2018/")                  # LightGBM baseline

# classify the carved unknown, statically (never executed)
with open("sample_4471.exe", "rb") as f:
    score = ember.predict_sample(model, f.read())
print(f"malware probability = {score:.3f}")              # -> 0.987
